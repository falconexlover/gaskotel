# Рекомендации по структуре заголовков в хедере

## Общие принципы

1. **Роль хедера**: Хедер сайта не должен содержать основной заголовок страницы (H1), так как основная цель хедера - навигация и брендинг.

2. **Семантическая разметка**:
   - Использовать тег `<header>` с `role="banner"`
   - Использовать тег `<nav>` с соответствующими `aria-label`
   - Для элементов навигации использовать теги `<a>` а не `<div>` или `<span>`

3. **Иерархия заголовков в хедере**:
   - Заголовки в хедере должны быть второстепенными (если вообще используются)
   - Основной H1 должен находиться в основном контенте страницы

## Реализация в компоненте Header

### Текущая структура:
```jsx
<header role="banner">
  <div role="complementary"> {/* Верхняя панель */ }
    {/* Информация о контактах и языках */}
  </div>
  
  <nav aria-label="Основная навигация">
    {/* Основное меню */}
  </nav>
  
  <nav aria-label="Мобильная навигация" role="navigation">
    {/* Мобильное меню */}
  </nav>
</header>
```

### ARIA атрибуты:
- `aria-label` - для описания назначения элементов
- `aria-expanded` - для состояния выпадающих меню
- `aria-haspopup` - для указания на наличие подменю
- `role="banner"` - для идентификации основного заголовка сайта
- `role="complementary"` - для вспомогательной информации в хедере

### Доступность:
1. Управление фокусом для клавиатурной навигации
2. Поддержка screen reader через ARIA-атрибуты
3. Соответствие стандартам WCAG 2.1

## Пример правильной иерархии на странице

```
<h1>Название страницы (находится в основном контенте)</h1>
<header role="banner">
  <nav aria-label="Основная навигация">
    <a href="/about">О заводе</a>
    <a href="/products">Продукция</a>
    ...
  </nav>
</header>
<main>
  <h1>Название страницы</h1> <!-- Только здесь должен быть H1 -->
  ...
</main>
```

## Проверка валидности

Для проверки правильности структуры можно использовать:
1. Инструменты разработчика браузера
2. axe-core для проверки доступности
3. HTML-валидаторы
4. Инструменты скринридеров

## Заключение

Хедер должен быть структурно логичным и доступным, но не должен включать основной заголовок страницы (H1). Основной H1 должен быть в основном содержании страницы, а хедер служит для навигации и брендинга.
